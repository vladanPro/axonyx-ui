Write pages that stay readable

Axonyx is aiming for the feeling React promised early on, but with a stronger full-stack shape, clearer file roles, and much less JavaScript by default.

The framework direction is simple: author in .ax, keep routes obvious, lower into Rust, and let CSS handle as much presentation as possible.

Build the kinds of sites we actually want

The first targets are framework sites, docs, marketing pages, and later CMS-driven products that should not need a thousand client-side scripts just to render content.

This repository is the proving ground for that direction.

Why Axonyx

Axonyx is being shaped as a Rust-first framework with authoring layers that stay readable, explicit, and easier to reason about than typical React-heavy stacks.

Minimal JS

The direction is CSS-first presentation, semantic HTML, and only as much JavaScript as the interaction actually needs.

AI-Friendly

Stable file roles, small numbers of correct patterns, and clear lowering stages should make Axonyx easier for both humans and AI agents.

Future UI Layer

The visual system should move into axonyx-ui, while axonyx-site stays the public showcase and real-world consumer of that layer.
